# theta_h264_ffmpeg_packet_pub

ROS 2 package that captures the H.264 stream from a **RICOH THETA** 360 camera through **GStreamer**, decodes it, re-encodes it with **FFmpeg**, and publishes it as `ffmpeg_image_transport_msgs/msg/FFMPEGPacket`.

The main executable is:

- `theta_h264_ffmpeg_packet_node`

This node is intended for low-latency video pipelines where the camera stream is published on a ROS 2 topic using the `ffmpeg` image transport, for example:

- input topic: `/camera/image_raw/ffmpeg`
- republished raw topic: `/camera/image_decoded`

## Overview

This package provides a custom ROS 2 node that:

1. Captures a RICOH THETA video stream using **GStreamer**
2. Uses the **gstthetauvc** GStreamer plugin to access the THETA UVC stream
3. Decodes the incoming H.264 bitstream
4. Re-encodes it using **FFmpeg**
5. Publishes encoded packets as `ffmpeg_image_transport_msgs/msg/FFMPEGPacket`

This makes it possible to stream video efficiently over ROS 2 with much less bandwidth than publishing raw `sensor_msgs/msg/Image` frames.

## Features

- ROS 2 node written in C++17
- GStreamer-based capture pipeline
- FFmpeg-based H.264 encoding
- Publishes `ffmpeg_image_transport_msgs/msg/FFMPEGPacket`
- Compatible with `ffmpeg_image_transport`
- Supports low-latency settings such as:
  - `preset=ultrafast`
  - `tune=zerolatency`
  - small GOP size
  - no B-frames

## Install dependencies

Install ROS/system dependencies first:

```bash
sudo apt update
sudo apt install -y \
  pkg-config \
  libgstreamer1.0-dev \
  libgstreamer-plugins-base1.0-dev \
  libavcodec-dev \
  libavutil-dev
```

Install the ROS 2 FFmpeg transport packages if needed:

```bash
sudo apt install -y \
  ros-${ROS_DISTRO}-ffmpeg-image-transport \
  ros-${ROS_DISTRO}-ffmpeg-image-transport-msgs
```

## Install gstthetauvc

Clone and build the THETA GStreamer plugin:

```bash
git clone https://github.com/nickel110/gstthetauvc.git
cd gstthetauvc/thetauvc
make
```

After building, make sure `gstthetauvc.so` is visible to GStreamer. Depending on your system, you may need to copy it into your local or system GStreamer plugin directory, or extend `GST_PLUGIN_PATH`.

Example:

```bash
export GST_PLUGIN_PATH=$GST_PLUGIN_PATH:/path/to/gstthetauvc/thetauvc
```

You can verify that GStreamer sees the plugin with:

```bash
gst-inspect-1.0 thetauvcsrc
```

## Build the package

Clone this repository into your ROS 2 workspace:

```bash
cd ~/ws/src
git clone <your-repository-url>
cd ~/ws
rosdep install --from-paths src --ignore-src -r -y
colcon build --packages-select theta_h264_ffmpeg_packet_pub
source install/setup.bash
```

## Run

Example command:

```bash
ros2 run theta_h264_ffmpeg_packet_pub theta_h264_ffmpeg_packet_node \
  --ros-args \
  -p camera_mode:=2K \
  -p encoder_name:=libx264 \
  -p bit_rate:=10000000 \
  -p preset:=ultrafast \
  -p tune:=zerolatency \
  -p gop_size:=10 \
  -p max_b_frames:=0
```

## Example pipeline

### 1. Publish encoded FFmpeg packets

```bash
ros2 run theta_h264_ffmpeg_packet_pub theta_h264_ffmpeg_packet_node \
  --ros-args \
  -p camera_mode:=4K \
  -p encoder_name:=libx264 \
  -p bit_rate:=24000000 \
  -p preset:=ultrafast \
  -p tune:=zerolatency \
  -p gop_size:=10 \
  -p max_b_frames:=0
```

### 2. Republish as raw images

```bash
ros2 run image_transport republish \
  --ros-args \
  --params-file repub_qos.yaml \
  --remap in/ffmpeg:=/camera/image_raw/ffmpeg \
  --remap out:=/camera/image_decoded
```

### 3. Visualize the decoded image

```bash
ros2 run rqt_image_view rqt_image_view --ros-args -p reliability:=best_effort
```

## Topics

### Published by this node

- `/camera/image_raw/ffmpeg`  
  Encoded FFmpeg packet stream published as `ffmpeg_image_transport_msgs/msg/FFMPEGPacket`

### Produced downstream

- `/camera/image_decoded`  
  Decoded raw image topic generated by `image_transport republish`

## Parameters

Typical parameters supported by the node:

- `camera_mode`  
  THETA camera mode, for example `2K` or `4K`

- `encoder_name`  
  FFmpeg encoder name, for example `libx264`

- `bit_rate`  
  Target H.264 bitrate in bits per second

- `preset`  
  FFmpeg encoder preset, for example `ultrafast`

- `tune`  
  FFmpeg encoder tune option, for example `zerolatency`

- `gop_size`  
  Distance between keyframes

- `max_b_frames`  
  Number of B-frames. Use `0` for lower latency

## Example republisher configuration

Example `repub_qos.yaml`:

```yaml
image_republisher:
  ros__parameters:
    in_transport: "ffmpeg"
    out_transport: "raw"
    qos_overrides:
      /camera/image_raw/ffmpeg:
        subscription:
          reliability: best_effort
          history: keep_last
          depth: 2
    in.ffmpeg.decoders.h264: "h264"
```

## Notes

- This package publishes **compressed video packets**, not raw `sensor_msgs/msg/Image` frames.
- Measuring the decoded topic with `ros2 topic hz` may reduce performance because it adds another full raw-image subscriber.
- For lower latency, prefer:
  - `preset:=ultrafast`
  - `tune:=zerolatency`
  - `gop_size:=10`
  - `max_b_frames:=0`

## Troubleshooting

### GStreamer cannot find `thetauvcsrc`

Make sure the `gstthetauvc` plugin has been built and that its location is included in `GST_PLUGIN_PATH`.

Check with:

```bash
gst-inspect-1.0 thetauvcsrc
```

### No decoded image is shown

Check:

- the packet publisher is running
- the topic `/camera/image_raw/ffmpeg` exists
- the republisher is subscribed to the correct input topic
- `ffmpeg_image_transport` is installed on the system

### Viewer lags when running `ros2 topic hz`

If you measure `/camera/image_decoded` with `ros2 topic hz`, the extra subscriber can reduce performance significantly because the decoded topic carries large raw images.

Prefer measuring the compressed topic instead:

```bash
ros2 topic hz /camera/image_raw/ffmpeg
```
